# CreditAI — Backend

FastAPI + PostgreSQL backend for the AI-powered alternative credit scoring system.

---

## Architecture

```
creditai-backend/
├── app/
│   ├── main.py                   ← FastAPI app, CORS, router registration
│   ├── core/
│   │   ├── config.py             ← Settings loaded from .env via pydantic-settings
│   │   └── security.py           ← JWT creation/verification, password hashing, auth deps
│   ├── db/
│   │   ├── base.py               ← Async SQLAlchemy engine + session factory
│   │   └── seed.py               ← Demo data (matches frontend dummy.ts exactly)
│   ├── models/                   ← SQLAlchemy ORM models
│   │   ├── user.py               → users table
│   │   ├── submission.py         → submissions table (one per credit application)
│   │   ├── document.py           → documents table (uploaded files + extraction)
│   │   ├── conversation.py       → conversation_messages table (agent chat log)
│   │   └── report.py             → credit_reports table (final scores + lender decisions)
│   ├── schemas/                  ← Pydantic request/response shapes
│   │   ├── auth.py
│   │   ├── submission.py
│   │   ├── agent.py
│   │   └── report.py
│   ├── services/
│   │   ├── agent_service.py      ← LLM extraction (Claude API) + gap-fill logic
│   │   ├── scoring_service.py    ← XGBoost inference + score→breakdown mapping
│   │   └── report_service.py     ← LLM report narrative + PDF generation
│   ├── ml/
│   │   ├── train_model.py        ← Train XGBoost on synthetic data
│   │   └── model.joblib          ← Saved model (generated by train_model.py)
│   └── api/routes/
│       ├── auth.py               ← POST /auth/signup, /auth/login, GET /auth/me
│       ├── submissions.py        ← Submission lifecycle + dashboard stats
│       ├── agent.py              ← Chat messages + conversation history
│       ├── reports.py            ← Generate, fetch, share, PDF download
│       ├── lender.py             ← Lender dashboard + decision recording
│       ├── history.py            ← Activity timeline
│       └── settings.py           ← Profile update
├── alembic/                      ← Database migrations
├── tests/
│   ├── test_auth.py
│   └── test_scoring.py
├── uploads/                      ← Uploaded files (local dev)
├── .env.example
├── alembic.ini
└── requirements.txt
```

## Quick Start

### Prerequisites
- Python 3.11+
- PostgreSQL running locally

### 1. Create and activate a virtual environment

```bash
cd creditai-backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment

```bash
cp .env.example .env
# Edit .env and fill in:
#   DATABASE_URL   – your PostgreSQL connection string
#   SECRET_KEY     – generate with: python -c "import secrets; print(secrets.token_hex(32))"
#   ANTHROPIC_API_KEY – your Claude API key from console.anthropic.com
```

### 4. Create database and run migrations

```bash
createdb creditai
alembic upgrade head
```

### 5. Train the ML model

```bash
python -m app.ml.train_model
```

This generates `app/ml/model.joblib`. You only need to do this once.

### 6. Seed demo data

```bash
python -m app.db.seed
```

Demo credentials after seeding:
- **Applicant**: `amara@example.com` / `demo1234`
- **Lender**: `lender@apexlending.com` / `lender1234`

### 7. Start the server

```bash
uvicorn app.main:app --reload --port 8000
```

Interactive API docs: http://localhost:8000/docs

## Database Schema

```
users
  id, name, phone (unique), country, email (unique, optional)
  hashed_password, role ("applicant"|"lender"), organization
  is_active, created_at, updated_at

submissions  (one per credit application)
  id, user_id → users
  status: pending → documents_uploaded → analyzing → awaiting_chat → scoring → completed
  extracted_features (JSON string of 12 ML features)
  created_at, updated_at

documents  (one row per uploaded file)
  id, submission_id → submissions
  original_filename, file_path, file_size_bytes, mime_type
  doc_type: "Utility Bill"|"Rent Receipt"|"Mobile Recharge History"|"Transaction Statement"
  extracted_text, extraction_confidence, is_low_confidence
  created_at

conversation_messages  (full audit log of agent chat)
  id, submission_id → submissions
  role: "ai"|"user"
  content, target_feature (which ML feature this AI turn was asking about)
  created_at

credit_reports  (one per completed submission)
  id, submission_id → submissions (unique), user_id → users
  score (0-100), risk_tier: low|medium|high, ml_probability
  score_breakdown, positive_signals, improvement_areas, report_text (all JSON strings)
  data_sources (JSON), model_version
  is_shared_with_lender, lender_decision, lender_notes, decision_at
  created_at
```

## API Reference

All endpoints are prefixed with `/api/v1`.

### Auth

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | `/auth/signup` | — | Register applicant → returns JWT |
| POST | `/auth/login` | — | Login with phone or email → returns JWT |
| GET | `/auth/me` | JWT | Get current user profile |

**Signup request:**
```json
{ "name": "Amara Osei", "phone": "+233241234567", "country": "Ghana",
  "email": "amara@example.com", "password": "mypassword" }
```

**Login request:**
```json
{ "identifier": "amara@example.com", "password": "mypassword" }
```

**Token response:**
```json
{ "access_token": "eyJ...", "token_type": "bearer",
  "user": { "id": 1, "name": "Amara Osei", "role": "applicant", ... } }
```

### Submissions & Documents

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | `/submissions/` | JWT | Create a new submission (returns id) |
| POST | `/submissions/{id}/documents` | JWT | Upload a file (`multipart/form-data`) |
| GET | `/submissions/dashboard` | JWT | All dashboard data in one call |
| GET | `/submissions/{id}` | JWT | Get submission + documents |

**Upload document** (`multipart/form-data`):
```
doc_type = "Utility Bill"
file     = <binary>
```

---

### AI Agent

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | `/agent/{submission_id}/message` | JWT | Send a message; get AI reply |
| GET | `/agent/{submission_id}/conversation` | JWT | Full message history |

**Message request:**
```json
{ "content": "Self-employed" }
```

**Message response:**
```json
{
  "id": 5, "role": "ai",
  "content": "How many people depend on your income?",
  "chips": ["0", "1-2", "3-5", "6+"],
  "created_at": "2026-03-09T10:28:00Z"
}
```

When the agent finishes all gap-fill questions it sets the submission status to `scoring`.
Call `/reports/{submission_id}/generate` immediately after to run the ML model.

## Full Applicant Flow

```
1. POST /auth/signup                         → get JWT
2. POST /submissions/                        → get submission_id
3. POST /submissions/{id}/documents          → upload file 1 (doc_type=Utility Bill)
4. POST /submissions/{id}/documents          → upload file 2 (doc_type=Mobile Recharge History)
5. POST /agent/{id}/message  { content: "" } → triggers LLM extraction
                                               agent replies with first question + chips
6. POST /agent/{id}/message  { content: "Self-employed" }
                                               → agent replies with next question
7. POST /agent/{id}/message  { content: "1-2" }
   ... repeat until agent says "Generating your score now..."
   (submission status becomes "scoring")
8. POST /reports/{id}/generate               → ML runs, report created
9. GET  /reports/{id}                        → render Report page
10. POST /reports/{id}/share                 → make visible to lenders
```

## Connecting the Frontend

In your frontend, replace all `@/data/dummy` imports with API calls.
Set the base URL in a single file, e.g. `src/lib/api.ts`:

```ts
export const API_BASE = "http://localhost:8000/api/v1";

export async function apiFetch(path: string, options?: RequestInit) {
  const token = localStorage.getItem("token");
  return fetch(`${API_BASE}${path}`, {
    ...options,
    headers: {
      "Content-Type": "application/json",
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
      ...options?.headers,
    },
  });
}
```

Key replacements:

| Frontend page | Replace dummy import with |
|---|---|
| `Signup.tsx` | `POST /auth/signup` → save token to localStorage |
| `Dashboard.tsx` | `GET /submissions/dashboard` |
| `Upload.tsx` | `POST /submissions/` then `POST /submissions/{id}/documents` |
| `Assistant.tsx` | `POST /agent/{id}/message` (poll for each reply) |
| `Report.tsx` | `GET /reports/latest` |
| `History.tsx` | `GET /history/` |
| `Settings.tsx` | `PATCH /settings/profile` |
| `Lender.tsx` | `GET /lender/applicants?search=&risk_tier=all` |
| `LenderReport.tsx` | `GET /lender/applicants/{report_id}` |
