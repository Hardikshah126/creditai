# CreditAI — Backend

Python + FastAPI backend for the CreditAI credit scoring platform.

---

## Stack

| Layer | Technology |
|---|---|
| API framework | FastAPI + Uvicorn |
| Database | PostgreSQL + SQLAlchemy ORM |
| Migrations | Alembic |
| Auth | JWT (python-jose) + bcrypt |
| LLM | Claude API (Anthropic) via `anthropic` SDK |
| ML Model | XGBoost (scikit-learn pipeline) |
| File handling | pypdf, Pillow, pytesseract |
| Validation | Pydantic v2 |

## Quick Start

### 1. Prerequisites

- Python 3.11+
- PostgreSQL 14+
- Tesseract OCR: `brew install tesseract` / `apt install tesseract-ocr`
- Anthropic API key

### 2. Setup

```bash
cd backend
bash setup.sh
```

This creates your venv, installs dependencies, copies `.env.example` to `.env`, and trains the ML model.

### 3. Configure

Edit `.env`:
```
DATABASE_URL=postgresql://youruser:yourpass@localhost:5432/creditai_db
ANTHROPIC_API_KEY=sk-ant-...
SECRET_KEY=generate-with-openssl-rand-hex-32
```

### 4. Database

```bash
# Create the database
createdb creditai_db

# Activate venv first
source venv/bin/activate

# Run migrations
alembic upgrade head
```

### 5. Run

```bash
uvicorn app.main:app --reload
```

API is live at `http://localhost:8000`
Swagger docs at `http://localhost:8000/docs`

## API Reference

### Auth

| Method | Path | Description |
|---|---|---|
| POST | `/api/v1/auth/signup` | Create applicant or lender account |
| POST | `/api/v1/auth/login` | Login with phone + password |
| GET | `/api/v1/auth/me` | Get current user |
| PUT | `/api/v1/auth/me` | Update profile |

### Submissions & Documents

| Method | Path | Description |
|---|---|---|
| POST | `/api/v1/submissions` | Create a new submission |
| GET | `/api/v1/submissions` | List my submissions |
| GET | `/api/v1/submissions/{id}` | Get one submission |
| POST | `/api/v1/submissions/{id}/documents` | Upload a document (multipart) |
| POST | `/api/v1/submissions/{id}/analyze` | Trigger LLM extraction |

### AI Chat

| Method | Path | Description |
|---|---|---|
| POST | `/api/v1/chat/start?submission_id=` | Start conversation |
| POST | `/api/v1/chat/reply` | Send user answer, get next question |
| GET | `/api/v1/chat/{id}/history` | Full conversation log |

### Reports

| Method | Path | Description |
|---|---|---|
| POST | `/api/v1/reports/{submission_id}/generate` | Run ML model, generate report |
| GET | `/api/v1/reports/my/latest` | My latest report |
| GET | `/api/v1/reports/my/history` | All my past reports |
| POST | `/api/v1/reports/{id}/share` | Generate share link |
| GET | `/api/v1/reports/shared/{token}` | Public report by token |
| GET | `/api/v1/reports/lender/applicants` | Lender: list shared applicants |
| GET | `/api/v1/reports/lender/{id}` | Lender: full report |
| POST | `/api/v1/reports/lender/decision` | Lender: record decision |

## Connecting the Frontend

### Step 1 — Copy integration files

```bash
cp backend/frontend_integration/api.ts       frontend/src/lib/api.ts
cp backend/frontend_integration/AuthContext.tsx  frontend/src/context/AuthContext.tsx
cp backend/frontend_integration/useApi.ts    frontend/src/hooks/useApi.ts
```

### Step 2 — Add env variable

In `frontend/.env.local`:
```
VITE_API_URL=http://localhost:8000/api/v1
```

### Step 3 — Wrap your app with AuthProvider

In `frontend/src/main.tsx`:
```tsx
import { AuthProvider } from "@/context/AuthContext";

ReactDOM.createRoot(document.getElementById("root")!).render(
  <AuthProvider>
    <App />
  </AuthProvider>
);
```

### Step 4 — Wire Signup page

Replace the dummy `navigate("/dashboard")` in `Signup.tsx`:
```tsx
import { useAuth } from "@/context/AuthContext";
import { useNavigate } from "react-router-dom";

const { signup } = useAuth();
const navigate = useNavigate();

const handleSubmit = async (e: React.FormEvent) => {
  e.preventDefault();
  try {
    await signup({ name, phone, country, password });
    navigate("/dashboard");
  } catch (err) {
    toast.error(err instanceof Error ? err.message : "Signup failed");
  }
};
```

### Step 5 — Wire Upload page

Replace static file handling with real API calls:
```tsx
import { useCreateSubmission, useUploadDocument, useAnalyzeDocuments } from "@/hooks/useApi";

const createSubmission = useCreateSubmission();
const uploadDoc = useUploadDocument();
const analyze = useAnalyzeDocuments();

// On page mount or first upload — create a submission
const [submissionId, setSubmissionId] = useState<string | null>(null);

const handleUpload = async (files: File[], docType: string) => {
  let sid = submissionId;
  if (!sid) {
    const sub = await createSubmission.mutateAsync();
    sid = sub.id;
    setSubmissionId(sid);
  }
  for (const file of files) {
    await uploadDoc.mutateAsync({ submissionId: sid, file, docType });
  }
};

const handleAnalyze = async () => {
  if (!submissionId) return;
  setAnalyzing(true);
  await analyze.mutateAsync(submissionId);
  navigate("/dashboard/assistant", { state: { submissionId } });
};
```

### Step 6 — Wire Assistant page

Replace hardcoded questions with real API:
```tsx
import { useStartChat, useSendMessage } from "@/hooks/useApi";
import { useLocation } from "react-router-dom";

const { state } = useLocation();
const submissionId = state?.submissionId;
const startChat = useStartChat();
const sendMessage = useSendMessage();
const generateReport = useGenerateReport();

// On mount
useEffect(() => {
  if (submissionId) {
    startChat.mutate(submissionId, {
      onSuccess: (data) => {
        setMessages([{ role: "ai", text: data.ai_message, chips: data.chips ?? undefined }]);
      },
    });
  }
}, [submissionId]);

// On user answer
const handleAnswer = async (answer: string) => {
  setMessages(prev => [...prev, { role: "user", text: answer }]);
  setTyping(true);

  const reply = await sendMessage.mutateAsync({ submissionId, message: answer });
  setTyping(false);
  setMessages(prev => [...prev, { role: "ai", text: reply.ai_message, chips: reply.chips ?? undefined }]);

  if (reply.is_complete) {
    // Generate the credit report
    const report = await generateReport.mutateAsync(submissionId);
    navigate("/dashboard/report");
  }
};
```

### Step 7 — Wire Report page

Replace dummy data with real API:
```tsx
import { useMyLatestReport, useShareReport } from "@/hooks/useApi";

const { data: report, isLoading } = useMyLatestReport();
const shareReport = useShareReport();

const handleShare = async () => {
  if (!report) return;
  const { share_url } = await shareReport.mutateAsync(report.id);
  await navigator.clipboard.writeText(share_url);
  toast.success("Link copied!");
};

// Use report.score, report.risk_tier, report.breakdown, etc.
```

### Step 8 — Wire Lender pages

```tsx
// Lender.tsx

const { data: applicants } = useLenderApplicants({ risk_tier: riskFilter, search });

// LenderReport.tsx

const { data: report } = useLenderReport(id!);

```

## The Full Request Flow

```
User uploads 2 docs
        │
        ▼
POST /submissions              → creates Submission row
POST /submissions/{id}/documents (x2) → saves files, creates AlternativeData rows
POST /submissions/{id}/analyze → Claude reads each doc, extracts features → stored as JSON
        │
        ▼
POST /chat/start               → agent checks missing features, sends first question
POST /chat/reply (x4)          → user answers, agent saves to AgentConversation
   [is_complete = true]
        │
        ▼
POST /reports/{id}/generate    → merges doc features + convo features
                               → XGBoost scores them (0-100)
                               → Claude writes narrative
                               → CreditReport row created
        │
        ▼
GET /reports/my/latest         → frontend displays score + breakdown
POST /reports/{id}/share       → generates share_token
GET /reports/lender/applicants → lender sees shared reports
POST /reports/lender/decision  → lender records approve/decline
```

## ML Model

The model ships with a synthetic training set for development.
To retrain on real data when you have it:

```bash
# Put your CSV at ml/training_data.csv with columns matching FEATURE_NAMES
# Then:
python ml/train_model.py
```

Model card is auto-generated at `ml/model_card.json` with AUC, feature importances, and version info.

## Environment Variables

| `DATABASE_URL` | local postgres | PostgreSQL connection string |
